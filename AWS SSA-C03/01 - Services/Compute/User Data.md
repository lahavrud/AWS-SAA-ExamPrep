כלי המאפשר להריץ סקריפטים של אוטומיזציה (Bootstrapping) ברגע שהשרת עולה בפעם הראשונה.

### נקודות מפתח לבחינה:

- **עיתוי:** הסקריפט רץ **פעם אחת בלבד** – בזמן ה-Boot הראשון של ה-Instance.
    
- **הרשאות:** ב-Linux, הסקריפט מבוצע כמשתמש **root** (אין צורך ב-`sudo`).
    
- **שימושים נפוצים:**
    
    - עדכון חבילות מערכת (`yum update`).
        
    - התקנת תוכנות (Apache, Docker, Python).
        
    - הורדת קבצים מ-S3.
        
    - הגדרת משתני סביבה.
        
- **מגבלות:** גודל ה-User Data מוגבל ל-**16KB**.
    
- **אבטחה:** הסקריפט נשמר כטקסט פשוט (Plaintext). **אסור** לשמור בו סיסמאות או Access Keys.

### איך ניגשים לזה מהשרת?

ניתן לראות את ה-User Data שהגדרת מתוך השרת עצמו על ידי פנייה לכתובת ה-Metadata:

```
curl http://169.254.169.254/latest/user-data
```

---

> [!warning] שים לב
> 
> אם תבצע Restart לשרת, ה-User Data לא ירוץ שוב. הוא מיועד להגדרה ראשונית בלבד.
