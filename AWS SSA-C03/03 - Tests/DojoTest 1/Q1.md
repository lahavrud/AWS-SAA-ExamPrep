# טעות: מיגרציה מ-On Premise Windows ל-AWS

> [!failure] השאלה בקצרה
> איך מבצעים מיגרציה לשרתי On Premise Windows כך שיהיו עם high availability ו-low latency ל-Block Storage

### מה אני בחרתי?
בחרתי ב-[[FSx for Windows]] כי קישרתי את זה בהכרח לWindows File System.

### למה זו טעות?
זו הטעיה: מערכת הקבצים של Windows עובדת עם קבצים, לא בלוקים (בלוקים - פרוטוקול iSCSI).

### התשובה הנכונה: [[FSx for NetApp ONTAP]]
* **הסבר:** מערכת הקבצים היחידה (ממה שלמדתי) שתומכת ב-Block Storage
* **נקודה קריטית למבחן:**  NetApp ONTAP היא מערכת קבצים שתומכת בפרוטוקולים רבים (Multiprotocol - תומכת: iSCSI, NFS, SMB)

### 💡 לקח לעתיד (Key Takeaway)
אם מופיעה בשאלה המילה **NetApp**, **ONTAP** או דרישה ל-**iSCSI** בסביבת ענן – זו כמעט תמיד התשובה הנכונה.
